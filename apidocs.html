Query Method :

     Let the MusicHub server is listening at port 1080, locally. The established server api search query url is http://127.0.0.1:1080/api/search. Queries are handled through urls only. There is no form to post and to get the results back. If a user searches for a (for example : "Overexposed"), append the API SQURL(API Search Query URL - API Squirell) with "?query=Overexposed" (The Squirell should look like "http://127.0.0.1:1080/api/search?query=Overexposed". If your database contains any song from the given query, the server returns a xml document. This is a basic query string. Obviously not always the user search is based on Album.The server by default queries the database based on album search.

     Let us make our squirells more specific that the server searches accordingly. To search the given keyword based on artist (example artist:"Maroon5"), the query string must be "?query=Maroon5&based=artist". Append that to the squirell. Now the Sqirell should look like "http://127.0.0.1:1080/api/search?query=Maroon5&based=artist" If you have any Marron5 based songs in your database it will return something.

     The XML file obtained is human readable. In "songs" tag, every "song" tag contains the song attributes.

  • title - Title of the song
  • album - Album of the song
  • artist - Artist of the song
  • year - the year when the song is released
  • url - Song URL in the server
    • The Song URL shoul be suffixed with "http://127.0.0.1:1080/" or (whatever address and the port MusicHub listens......) to get the mp3 resource.
